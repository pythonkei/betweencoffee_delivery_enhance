# 支付回調重定向問題修復報告

**報告日期**: 2026年3月14日  
**問題編號**: PAYMENT-REDIRECT-001  
**狀態**: ✅ 已修復  
**影響範圍**: 支付寶支付回調流程

## 📋 問題描述

### 問題現象
客戶使用支付寶完成支付後，返回網站時無法正確重定向到支付確認頁面，而是被重定向到首頁。

### 影響
1. 用戶無法看到支付成功的確認信息
2. 用戶體驗受損，可能導致用戶重複支付
3. 系統無法正確記錄支付完成狀態

## 🔍 根本原因分析

### 1. 訂單不存在時的錯誤處理問題
在 `order_payment_confirmation` 視圖中，當訂單不存在時，原本的代碼會調用 `handle_order_error` 函數，該函數會嘗試重定向到 `eshop:order_detail` 頁面。但由於訂單不存在，無法提供有效的 `order_id` 參數，導致重定向失敗，最終回退到首頁。

### 2. 錯誤處理邏輯不完善
`handle_order_error` 函數在重定向失敗時會回退到首頁，而不是顯示一個友好的錯誤頁面。

### 3. 缺乏調試日誌
支付回調過程中缺乏足夠的調試日誌，難以追蹤重定向路徑。

## 🛠️ 實施的修復

### 1. 修復 `order_payment_confirmation` 視圖
**檔案**: `eshop/views/order_views.py`
**修改內容**:
```python
# 原代碼:
except OrderModel.DoesNotExist:
    return handle_order_error(
        request,
        f'訂單 #{order_id} 不存在',
        redirect_url='eshop:order_detail',  # 改為訂單詳情（但訂單不存在，可能需要跳轉到列表）
        error_type='order'
    )

# 修復後代碼:
except OrderModel.DoesNotExist:
    # 訂單不存在，顯示錯誤信息但不重定向到首頁
    logger.error(f"訂單 #{order_id} 不存在，但用戶從支付回調訪問")
    
    # 創建一個友好的錯誤頁面上下文
    context = {
        'payment_status': 'error',
        'error_message': f'訂單 #{order_id} 不存在或已被處理',
        'order_id': order_id,
        'suggested_action': '請聯繫客服或查看您的訂單歷史'
    }
    
    # 記錄詳細信息用於調試
    logger.info(f"支付回調訪問不存在的訂單: {order_id}")
    logger.info(f"請求路徑: {request.path}")
    logger.info(f"請求參數: {request.GET}")
    
    return render(request, 'eshop/order_payment_confirmation.html', context)
```

### 2. 增強 `alipay_callback` 函數的調試日誌
**檔案**: `eshop/views/payment_views.py`
**修改內容**:
- 添加詳細的訂單處理日誌
- 記錄重定向前的狀態信息
- 記錄重定向目標URL

### 3. 修復語法錯誤
修復了 `alipay_callback` 函數中的語法錯誤（try 語句缺少對應的 except/finally）。
