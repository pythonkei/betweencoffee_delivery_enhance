# Between Coffee 系統 .clinerules

## 項目概述
- **項目名稱**: Between Coffee 咖啡店外帶網站與訂單製作管理系統
- **技術棧**: Django + PostgreSQL + WebSocket + Bootstrap 5
- **核心業務流程**: 顧客下單 → 支付處理 → 加入製作隊列 → 員工製作 → 訂單就緒 → 顧客取餐
- **部署平台**: Railway
- **當前狀態**: 系統穩定，核心功能完整，已完成系統架構清理、訂單狀態卡片修復和支付確認頁面優化

## 開發規範

### 代碼質量要求
1. **文檔標準**: 所有新函數必須附帶詳細文檔
   - Python: 使用 docstring 格式（Google 風格）
   - JavaScript: 使用 JSDoc 風格註釋
   - 文檔必須包含參數說明、返回值說明和示例

2. **函數設計**: 
   - 函數必須有明確的回傳值
   - 避免出現沒有傳回的函數
   - 單一職責原則：每個函數只做一件事

3. **測試要求**:
   - 所有新增功能必須覆蓋至少 80% 的單元測試
   - 關鍵業務邏輯必須有集成測試
   - 測試用例必須包含邊界情況和錯誤處理

### 安全要求
1. **身份驗證與授權**:
   - WebSocket 連接必須經過正確的身份驗證和授權
   - 使用 django-allauth 處理社交登入（Google、Facebook）
   - 敏感操作必須有權限檢查

2. **數據安全**:
   - 敏感數據（支付信息、用戶資料）必須加密存儲
   - 所有用戶輸入必須進行驗證和清理
   - 防止 SQL 注入、XSS、CSRF 攻擊

3. **支付安全**:
   - 支付處理必須使用安全的支付網關（PayPal、Alipay、FPS）
   - 支付交易必須有完整的日誌記錄
   - 支付狀態必須實時同步

### 性能優化要求
1. **數據庫優化**:
   - 所有查詢必須使用適當的索引
   - 避免 N+1 查詢問題
   - 使用查詢緩存和結果緩存

2. **WebSocket 優化**:
   - 實現指數退避重連策略
   - 連接狀態必須有監控和告警
   - 消息傳輸必須高效且可靠

3. **前端優化**:
   - 資源必須進行懶加載和壓縮
   - 確保移動端響應式設計
   - 減少不必要的 DOM 操作

## 項目特定規則

### Django 開發規範
1. **架構遵循**:
   - 使用 Django Channels 處理 WebSocket 連接
   - 遵循 Django REST Framework 最佳實踐
   - 使用 Django 的內置安全功能

2. **模型設計**:
   - 使用 Django 的 ORM 進行數據庫操作
   - 模型必須有適當的索引和約束
   - 使用 Django 的遷移系統管理數據庫變更

3. **視圖設計**:
   - 視圖必須遵循 DRY 原則
   - 業務邏輯應該放在服務層或工具函數中
   - 錯誤處理必須統一且友好

### 前端開發規範
1. **UI/UX 標準**:
   - 使用 Bootstrap 5 進行響應式設計
   - 確保移動端體驗優先
   - 保持一致的設計語言和用戶體驗

2. **JavaScript 開發**:
   - 使用模塊化的 JavaScript 代碼
   - 避免全局變量污染
   - 使用現代 JavaScript 特性（ES6+）

3. **實時功能**:
   - 使用 WebSocket 實現實時狀態更新
   - 狀態管理必須一致且可靠
   - 用戶界面必須反映最新的系統狀態

### 部署與維護
1. **部署流程**:
   - 使用 Railway 進行自動化部署
   - 部署前必須通過所有測試
   - 必須有回滾計劃

2. **監控與日誌**:
   - 系統必須有完整的監控和日誌記錄
   - 錯誤必須有適當的告警機制
   - 性能指標必須定期檢查

3. **文檔維護**:
   - 所有變更必須更新相關文檔
   - 文檔必須保持最新且準確
   - 必須有系統架構文檔和 API 文檔

# Between Coffee 系統 - 上下文摘要

**版本**: v2.0+

## 技術棧
- **backend**: Django 4.x, Django Channels, PostgreSQL, Redis
- **frontend**: HTML5, CSS3, JavaScript, Bootstrap 5, jQuery
- **deployment**: Railway, Docker (可選)
- **payments**: PayPal, Alipay, FPS, 現金支付
- **authentication**: django-allauth, Google OAuth, Facebook OAuth

## 核心模塊
- **eshop**: 主要業務邏輯模塊（商品、訂單、支付、隊列）
- **cart**: 購物車功能模塊
- **socialuser**: 用戶管理模塊（社交登入、用戶資料）
- **restaurant**: 餐廳管理模塊（員工、權限）
