3. 訊息類型測試

測試不同的 WebSocket 訊息類型：

舊格式：order_update

{
  "type": "order_update",
  "update_type": "status",
  "data": {
    "status": "preparing",
    "status_display": "製作中"
  }
}

新格式：order_status

{
  "type": "order_status",
  "data": {
    "status": "preparing",
    "status_display": "製作中",
    "queue_position": 2,
    "estimated_time": "14:50",
    "progress_percentage": 60
  }
}

專門更新：queue_position

{
  "type": "queue_position",
  "position": 1,
  "estimated_time": "14:48"
}

4. 修復總結

問題分析

  • 問題：訂單詳情頁面進度條無法實時更新
  • 根本原因：WebSocket 訊息類型不匹配
  • 前端期望：type: 'order_update'
  • 後端發送：type: 'order_status'
  • 影響：JavaScript 無法處理後端發送的狀態更新

修復內容

  • 修復文件：static/js/staff-order-management/order-detail.js
  • 修復位置：handleMessage 方法
  • 修復前：只處理 order_update 類型
  • 修復後：同時處理 order_status、queue_position、estimated_time 類型
  • 兼容性：保持對舊格式的支持

預期效果

  • ✅ WebSocket 連線正常建立
  • ✅ 能夠處理後端發送的 order_status 訊息
  • ✅ 進度條實時更新
  • ✅ 時間軸狀態自動更新
  • ✅ 隊列位置和預計時間顯示正確
  • ✅ 無需手動刷新頁面
