# 客戶訂單頁面自動更新修復報告

## 📅 修復日期：2026-03-19

---

## 🎯 問題描述

客戶在訂單詳情頁面 `http://localhost:8081/eshop/order/detail/` 無法自動更新訂單狀態的 4 個時間軸狀態，需要手動按 F5 刷新才能看到最新狀態。

---

## 🔍 問題根因分析

經過全面代碼分析，發現了以下問題：

### 問題 1：WebSocket 消息類型不匹配

**後端發送的消息**（`order_status_manager.py`）：
```python
send_order_update(
    order_id=order_id,
    update_type='status_change',  # ← 發送 'status_change'
    data={...}
)
```

**前端期望接收的消息**（`order-detail.js`）：
```javascript
switch (data.update_type) {
    case 'status':  // ← 只處理 'status'
        this.updateOrderStatus(data.data);
        break;
}
```

**結果**：後端發送 `'status_change'`，前端只處理 `'status'`，導致消息被忽略。

### 問題 2：Channel Layer 消息路由缺失

`websocket_utils.py` 中的 `broadcast_to_group` 發送消息時使用：
```python
message = {
    'type': 'order_update',  # channel layer 的 type
    ...
}
```

但 `consumers.py` 中的 `OrderConsumer` 沒有定義 `order_update` 方法，只有 `order_status_update` 方法，導致消息無法正確路由。

---

## ✅ 修復方案

### 修復 1：`eshop/consumers.py` - 新增消息路由方法

新增 `order_update` 方法作為消息路由器，根據 `update_type` 分發到對應的處理方法：

```python
async def order_update(self, event):
    """
    訂單更新路由器（來自 websocket_utils.send_order_update）
    """
    update_type = event.get('update_type', '')
    
    if update_type in ('status', 'status_change'):
        await self.order_status_update(event)
    elif update_type == 'payment_status':
        await self.payment_status_update(event)
    # ... 其他類型
```

### 修復 2：`eshop/order_status_manager.py` - 統一消息類型

將 `update_type` 從 `'status_change'` 修改為 `'status'`：

```python
send_order_update(
    order_id=order_id,
    update_type='status',  # ← 修復：改為 'status'
    data={
        'status': new_status,
        'status_display': order.get_status_display(),
        'message': f"訂單狀態已更新為 {new_status}"
    }
)
```

### 修復 3：`static/js/staff-order-management/order-detail.js` - 增強消息處理

擴展 `handleMessage` 方法，支持更多消息類型：

```javascript
// 同時支持 'status' 和 'status_change'
case 'status':
case 'status_change':
    this.updateOrderStatus(data.data);
    break;

// 新增 order_status 類型支持
else if (data.type === 'order_status') {
    this.updateOrderStatus(data.data);
}

// 新增 order_ready 類型支持
else if (data.type === 'order_ready') {
    this.showToast('🎉 您的訂單已準備就緒！', 'success');
    this.updateOrderStatus({ status: 'ready' });
}
```

## 📊 訂單狀態流程確認

您的理解是**正確的**：

| 狀態 | 觸發點 | 員工操作 |
|------|--------|----------|
| **已下單** | 訂單建立且付款成功 | 自動觸發 |
| **製作中** | waiting 渲染器，點擊「開始製作」 | `startPreparation(orderId)` |
| **待取餐** | preparing 渲染器，點擊「已就緒」 | `markAsReady(orderId)` |
| **已完成** | ready 渲染器，點擊「客戶已提取」 | `markAsCollected(orderId)` |
