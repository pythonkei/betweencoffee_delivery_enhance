## 2. 核心業務流程

### 顧客端下單流程（完整流程）

1. **商品瀏覽**
   - 訪問首頁 (`/`)
   - 瀏覽咖啡/咖啡豆商品
   - 查看商品詳情（圖片、價格、描述）

2. **購物車管理**
   - 添加商品到購物車
   - 選擇規格選項：
     - 咖啡：杯型（Small/Medium/Large）、牛奶量（Light/Medium/Extra）
     - 咖啡豆：重量（200g/500g）、研磨度（Non/Light/Medium/Deep）
   - 調整數量、移除商品

3. **結帳流程**
   - 進入結帳頁面 (`/eshop/order/confirm/`)
   - 填寫聯繫信息：姓名、電話、郵箱（可選）
   - 選擇取貨時間：5/10/15/20/30分鐘後
   - 選擇訂單類型：普通訂單 / 快速訂單（優先處理）

4. **支付處理**
   - 選擇支付方式：
     - 支付寶（線上支付）
     - PayPal（國際支付）
     - FPS轉數快（香港本地支付）
     - 現金支付（到店支付）
   - 完成支付流程
   - 支付成功後自動清空購物車

5. **訂單確認**
   - 生成4位取餐碼（唯一）
   - 生成二維碼（Base64編碼）
   - 顯示訂單詳情頁面
   - 提供訂單追蹤鏈接

6. **訂單追蹤**
   - 實時狀態更新（WebSocket）
   - 進度條顯示（0% → 100%）
   - 隊列位置顯示
   - 預計取貨時間

### 員工端製作流程（後台管理）

1. **隊列監控**
   - 登錄員工管理頁面 (`/eshop/queue/staff-management/`)
   - 查看實時隊列儀表板
   - 監控各狀態訂單數量

2. **訂單處理**
   - **等待中訂單**：查看訂單詳情，點擊"開始制作"
   - **製作中訂單**：顯示製作進度，點擊"標記就緒"
   - **已就緒訂單**：等待顧客提取，可手動標記為已提取

3. **狀態管理**
   - waiting → preparing（開始製作）
   - preparing → ready（製作完成）
   - ready → completed（顧客提取）

4. **異常處理**
   - 訂單取消（支付超時、顧客取消）
   - 訂單修改（規格變更）
   - 隊列修復（數據不一致）

## 3. 後端邏輯功能

### 核心模型定義

#### OrderModel（訂單模型）
- **狀態管理**：`pending` → `waiting` → `preparing` → `ready` → `completed`
- **支付狀態**：`pending` → `paid` → `cancelled` → `expired`
- **關鍵字段**：
  - `pickup_code`：4位取餐碼（唯一）
  - `qr_code`：二維碼數據（Base64）
  - `estimated_ready_time`：預計就緒時間
  - `payment_method`：支付方式（alipay/paypal/fps/cash）
  - `items`：JSON格式商品列表

#### CoffeeQueue（咖啡隊列）
- **隊列狀態**：`waiting` → `preparing` → `ready` → `completed`
- **關鍵字段**：
  - `position`：隊列位置（0表示不在隊列中）
  - `coffee_count`：咖啡杯數
  - `preparation_time_minutes`：預計製作時間
  - `estimated_completion_time`：預計完成時間

### 核心管理類

#### OrderStatusManager（訂單狀態管理器）
```python
# 主要功能方法
1. process_payment_success()      # 處理支付成功邏輯
2. process_order_status_change()  # 處理狀態變化
3. mark_as_preparing_manually()   # 手動標記為製作中
4. mark_as_ready_manually()       # 手動標記為就緒
5. analyze_order_type()           # 分析訂單類型
6. should_add_to_queue()          # 判斷是否加入隊列
```

#### CoffeeQueueManager（隊列管理器）
```python
# 主要功能方法
1. add_order_to_queue()           # 添加訂單到隊列
2. calculate_preparation_time()   # 計算製作時間
3. update_estimated_times()       # 更新預計時間
4. recalculate_all_order_times()  # 重新計算所有時間
5. mark_as_ready()                # 標記訂單為就緒
```

### 統一時間服務

#### 時間計算邏輯
- **基礎製作時間**：5分鐘（第一杯咖啡）
- **額外製作時間**：每增加一杯 +3分鐘
- **純咖啡豆訂單**：無需製作時間
- **快速訂單**：優先處理，時間計算不同

#### 時間格式化
- **香港時區**：Asia/Hong_Kong (UTC+8)
- **顯示格式**：
  - 完整格式：`2026-02-22 21:30:00`
  - 簡短格式：`21:30`
  - 相對時間：`5分鐘後`

## 5. URL配置結構

### 主路由配置 (`eshop/urls.py`)
```python
urlpatterns = [
    # 訂單相關路由
    path('order/', include('eshop.urls_order')),
    
    # 支付相關路由
    path('payment/', include('eshop.urls_payment')),
    
    # 隊列相關路由
    path('queue/', include('eshop.urls_queue')),
    
    # 員工管理路由
    path('staff/', include('eshop.urls_staff')),
    
    # API路由（用於實時更新等）
    path('api/', include('eshop.urls_api')),
]
```

### 隊列路由詳解 (`eshop/urls_queue.py`)

#### 頁面路由
- `/eshop/queue/dashboard/` - 隊列儀表板（統計視圖）
- `/eshop/queue/management/` - 隊列管理（詳細視圖）
- `/eshop/queue/staff-management/` - 員工訂單管理（操作視圖）

#### 數據API路由
- `/eshop/queue/unified-data/` - **統一數據API**（核心端點）
- `/eshop/queue/order-details/<order_id>/` - 訂單詳情API

#### 操作API路由
- `/eshop/queue/start/<order_id>/` - 開始製作API
- `/eshop/queue/ready/<order_id>/` - 標記就緒API
- `/eshop/queue/collected/<order_id>/` - 標記已提取API

#### 維護API路由
- `/eshop/queue/repair/` - 隊列數據修復
- `/eshop/queue/force-sync/` - 強制同步
- `/eshop/queue/cleanup/` - 數據清理

### 統一數據API響應格式
```json
{
    "success": true,
    "data": {
        "waiting_orders": [...],      // 等待隊列數據
        "preparing_orders": [...],    // 製作中隊列數據  
        "ready_orders": [...],        // 已就緒訂單數據
        "badge_summary": {            // 徽章數據
            "waiting": 5,
            "preparing": 3,
            "ready": 2,
            "completed": 10
        }
    },
    "timestamp": "2026-02-22T21:30:00+08:00",
    "timezone": "Asia/Hong_Kong",
    "message": "數據加載成功"
}
```

### 訂單狀態流程圖
```
顧客下單 → 支付處理 → 訂單確認 → 加入隊列 → 開始製作 → 製作完成 → 顧客提取
   ↓         ↓         ↓         ↓         ↓         ↓         ↓
 pending   pending   waiting   waiting   preparing  ready    completed
```

### 隊列管理流程圖
```
訂單支付成功
    ↓
檢查訂單類型（咖啡/咖啡豆）
    ↓
咖啡訂單 → 加入CoffeeQueue（status='waiting'）
    ↓
計算隊列位置和預計時間
    ↓
咖啡師點擊"開始制作" → status='preparing'
    ↓
咖啡師點擊"標記就緒" → status='ready'
    ↓
顧客提取 → status='completed'
    ↓
清理隊列位置（position=0）
```

### 支付流程圖
```
選擇支付方式
    ↓
支付寶/PayPal/FPS/現金
    ↓
支付處理
    ↓
支付成功回調
    ↓
更新訂單狀態（payment_status='paid'）
    ↓
觸發相關事件（清空購物車、發送通知）
    ↓
加入製作隊列（如果包含咖啡）
```
